## 🎨 设计系统基础提示词

### 整体风格提示词
```
企业级VM监控系统界面设计，采用深色主题为主，专业监控仪表板风格。整体风格简洁、专业、数据导向，参考Prometheus、Grafana等监控系统的设计语言，同时融入现代企业管理系统的视觉特征。

关键特征：
- 深色背景（#0f1419为主色调）减少视觉疲劳
- 数据可视化为主，图表和指标突出显示
- 信息层级清晰，重要数据醒目
- 色彩系统用于状态区分（正常=绿色，警告=橙色，危险=红色）
- 适合长时间观看的护眼设计
- 支持多语言（英文、中文、日文）
- 响应式设计，支持桌面端和移动端
```

```
- 错误提示区域（明显但不过度干扰）

- 深色科技感背景（#0f1419）

- 手势操作支持

- 桌面：1024px+（卡片固定500px宽）

错误处理：
- 登录失败时的震动反馈
- 账户锁定时的倒计时显示
- 密码重置流程入口
- 网络异常的重试机制

无障碍设计：
- 完整的表单标签关联
- 屏幕阅读器支持
- 高对比度模式适配
- 键盘完整导航
```

```
- 操作按钮：编辑、重置密码、启用/禁用、查看权限

用户编辑模态框：
- 基本信息：姓名、邮箱、部门、电话
- 角色分配：多选角色选择器，显示角色描述
- 状态管理：启用/禁用、设置过期时间

- 基本信息：角色名称、描述、父角色选择
- 权限设置：功能级权限选择器
- 用户列表：当前角色下的所有用户
- 实时权限预览：显示最终有效权限（合并继承权限）

**权限矩阵标签页**：
权限矩阵表格：
- 行：角色名称（按层级排序）
- 列：功能模块（监控数据查看、告警管理、系统配置、用户管理）
- 单元格：权限级别标签（无权限、只读、编辑、管理）
- 支持批量权限设置：选择多行设置相同权限

权限详情面板：
- 点击单元格显示该角色在此模块的详细权限
- 显示权限来源：继承权限 vs 直接授予
- 权限级别说明：每个级别允许的具体操作

**权限管理逻辑**：
- 只显示最终有效权限，隐藏继承关系
- 权限冲突在配置时实时检测并阻止保存
- 权限变更后立即生效，无需重新登录
- 支持权限变更历史追踪

**移动端适配**：
- 用户列表使用卡片式布局（每张卡片一个用户）
- 角色管理使用折叠列表，点击展开子角色
- 权限矩阵使用可滚动的表格，支持横向滑动
- 编辑操作使用全屏模态框
- 重要操作有二次确认弹窗（Toast提示）

**特殊功能**：
- 权限模板：快速创建常用角色权限组合
- 权限审计：显示权限变更历史和操作者
- 紧急权限：临时提升权限（有时间限制）
- 权限报告：生成用户权限汇总报告

**安全考虑**：
- 敏感操作需要二次确认
- 权限变更操作日志记录
- 支持权限冻结（紧急情况）
```

## 🌐 多语言设计提示词

### 多语言适配原则
```
设计原则：
- 文本容器自适应高度（适应不同语言长度）
- 日文和中文使用更大的行距（1.6-1.8）
- 英文保持标准行距（1.5）
- 数字和单位使用统一格式
- 按钮和容器最小宽度考虑文本长度

文本长度规划：
- 英文文本基准：100%长度
- 中文翻译：通常短20-30%
- 日文翻译：通常长10-20%
- 设计时预留30%的文本空间

UI适配考虑：
- 按钮和容器最小宽度考虑文本长度
- 表单标签右对齐（中/日文）vs 左对齐（英文）
- 表格列宽自适应不同语言文本
- 面包屑导航文本截断处理
```

## ⚠️ 重要提醒

1. **文本空间预留**: 设计时预留30%的文本空间，特别要考虑日文文本可能比英文长10-20%的情况。

2. **多语言测试**: 每个页面都要测试三种语言的显示效果，确保布局不会因为文本长度变化而破坏。

3. **响应式优先**: 移动端适配是重点，确保所有功能在手机上都能正常使用。

4. **权限复杂性**: 用户管理页面的权限展示要简化，隐藏继承关系的复杂性。

5. **数据可视化**: 监控数据的图表要清晰易读，支持实时更新和交互操作。
